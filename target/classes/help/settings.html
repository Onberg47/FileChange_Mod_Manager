Settings

Settings are important!

Normal settings

Every user is expected to at least look at these settings. Ideally you should set them!

Default Mod storage directory

The auto-populated path when adding a new game. This is if you want to store your mods in a central location. (Recommended)

For example if you wanted to do:

Documents/
    Mods/
        cyberpunk/
        fallout-4/
        ghost-recon/

Then make this setting Documents/Mods so when you add a new game you can just add the game-specific directory.

Trash warning

  • (Full) Will warn you with a popup dialog prompting to clean when your trash size exceeds your limit.
  • (Light) Only a warning in the console will be logged when size is exceeded.
  • (Off) No warning or checks on trash. (default)

Advanced settings

Changing these is for power users. It won't break everything but you must understand why you want to change it.

Deployment Manager directory

This is where, inside the Game installation path, the Mod Manager will store all it's deployment files.

It won't break anything (expect all previous ModManager data in Games) Only set this if there is a conflict! (If a directory called .mod_manager already exists in your game!?)

Expert settings (do not touch!)

These will break the program if incorrect and are not for regular users! Changing these is only for very rare cases where you would want to re-organise key-parts of the program itself.

The only useful scenario I can think of, is to store the temp and/or trash directories on the same physical storage devices as your games. (An idea with merit...)

Due to this, if you are curious about these settings, read the technical documentation on the program first. This user-manual will not cover them.

These settings may be moved off the GUI in future.

Trash management

In the settings page is also the Trash manager. This is the only source of all true delete operations, if its deleted here, its gone!

The manager offers:

  • Displaying how many Megabytes of actual disk-space the trash currently is using.
  • A percentage read-out of how close the trash space used is to your set limit.
  • Setting a limit on the overall disk-size of trash.
  • Setting a limit on the maximum age (days old) files are to qualify for cleaning.
  • The ability to Clean or Empty trash.
  • A shortcut to open the trash directory in the default directory application (file manager usually)

Cleaning your trash

Cleaning is not done automatically to ensure data is retained unless explicit deleted.

When a clean is performed it will delete all files older than the specified number of days. If the total size still exceeds the limit, starting from the oldest, files will be deleted until the maximum size is reached.

Emptying your trash

The entire trash directory will simply be nuked.
